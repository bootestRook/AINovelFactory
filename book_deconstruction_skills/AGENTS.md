# 拆书 Agent 共享硬契约

所有拆书 Agent 都必须遵守本契约；单个 `SKILL.md` 只补充本节点职责，不覆盖这里的底线。

## 证据
- 只能依据当前 prompt 给出的章节文本和上游产物。
- 不用系列外知识补剧情、人物、地点、组织、物品或设定。
- 结论必须带证据线索；证据不足就写“证据不足”，不要写成事实。

## 完整性
- 处理对象必须覆盖当前 prompt 给出的全部章节和全部上游可用产物。
- 如果 token 或文本截断导致无法完整覆盖，必须在输出开头写明缺口，不要假装已完整分析。
- 列表类结果不得只展开前几项；无法展开长档案时，先给全量短档案。

## 可复用输出
- 输出必须使用稳定 Markdown 标题，便于后续解析。
- 需要索引、档案、模板、规则或文件内容时，必须给实际条目，不要只写总结。
- 不要声称生成了某个 JSON、文件或 Skill，除非正文中已经给出可直接保存的内容。

## 复制风险
- 可抽象方法，不保留原书专名、桥段组合、人物替身、长句结构或可直接复刻的设定。
- 模板和写作 Skill 只输出可迁移规则，不输出改名版原书剧情。
