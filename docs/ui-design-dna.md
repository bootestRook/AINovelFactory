# UI Design DNA

本项目的 UI 基准来自当前主界面。新增或修改前端界面时，优先继承这里的规则，不另起一套视觉语言。

## 设计系统

- 色彩：冷白背景 `#FAFAFA`，白色卡片 `#FFFFFF`，细灰边框 `#E5E7EB`，深色正文 `#111827`，灰色辅助文字 `#6B7280`，红色只作品牌点缀和少量状态强调。
- 暗色：背景 `#0F1115`，卡片 `#171A21`，边框 `#2A2F3A`，正文 `#F5F7FB`，辅助文字 `#9AA3B2`。
- 字体：使用项目默认 `Microsoft YaHei UI`，标题加粗但不过度放大，正文保持紧凑清晰。
- 圆角：卡片 12px，按钮和输入框 8-10px。不要使用大圆角胶囊式弹窗面板。
- 边框与阴影：主要靠 1px 细边框建立层级，只使用很轻的阴影。禁止厚重投影、彩色发光、玻璃拟态。
- 布局：内容居中，最大宽度约 1224px；桌面首屏采用左主卡 + 右统计卡的严格网格；模块间留白克制。

## 组件规则

- 卡片：白底、细边框、轻阴影；卡片内部不要再嵌套装饰卡。
- 弹窗：白底、12px 圆角、细边框/轻阴影，内容按主界面卡片风格组织；不要使用粉色、米色或额外主题色背景。
- 输入框：使用框型输入框，不使用单条下划线输入；边框默认灰色，聚焦时用品牌红。
- 下拉框：复用项目统一下拉/输入框样式，最多展示 4 个选项，其余滚动并显示滚动条；不要铺满屏幕、不要另起偏色菜单面板。
- 按钮：主按钮深色底白字，次按钮白底灰边；红色不作为大面积按钮底色。
- 图标：使用 Flutter/Material 图标即可，尺寸 18-24px，作为功能提示而非装饰。

## 风格约束

- 整体气质：安静、工具化、清晰、克制，适合长时间写作。
- 禁止：营销页式大渐变、粉色/米色整块表单、夸张圆角、强装饰插画、全屏长下拉、单独发明一套弹窗风格。
- 新界面应先复用 `AppTheme`、`AppPalette` 和主界面已有组件模式；只有主界面规则无法覆盖时才补充新样式。

## 布局验收规则

- 所有 UI 布局改动必须截图验证后才能交付。
- 截图至少覆盖桌面默认窗口尺寸；涉及弹窗、下拉、滚动列表、空状态、暗色模式时，也要覆盖对应状态。
- 验证重点：文本不溢出，按钮文字不挤压，卡片不互相覆盖，弹窗内容不裁切，下拉不铺满屏幕，滚动条可见且可用。
- 禁止交付任何控件重叠、文本遮挡、内容错位、不可见操作区。

## 设计沉淀规则

- 用户明确验收通过的 UI 设计要素，必须补充到本文档。
- 补充时先判断适用范围：
  - `通用`：影响全局色彩、字体、圆角、间距、阴影、交互密度。
  - `组件`：只影响按钮、输入框、弹窗、下拉、卡片、列表、导航等组件。
  - `功能区`：只影响主界面、项目详情、写作编辑器、素材管理、AI 配置等具体区域。
  - `禁用`：明确不要再出现的风格、布局或交互。
- 只沉淀稳定规则，不记录未验收方案、一次性页面内容、临时实现细节。

## 功能区规则

- 主界面：沿用当前“最近写作/开始创作 + 今日目标 + 项目列表”的桌面工作台布局，保持居中内容宽度、左主右辅、下方项目列表的层级。

## 禁用规则

- 不要为了单个新功能新增一套独立视觉风格；先继承本文档和主界面模式。
