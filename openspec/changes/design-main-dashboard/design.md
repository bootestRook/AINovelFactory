## Context

当前仓库还没有应用主体代码或既有 OpenSpec specs。本变更先定义主界面计划，供后续 Flutter Windows 桌面实现使用。

参考 UI 是一个轻量创作工作台：顶部应用栏、欢迎区、左侧主任务卡、右侧统计卡、下方项目列表。项目要求是复用这种布局气质，但所有业务内容必须来自真实本地数据，不能用示例小说、假封面、假统计或假配置撑版面。

## Goals / Non-Goals

**Goals:**

- 建立主界面的信息架构：应用栏、欢迎区、主卡片、统计区、小说项目区。
- 明确无小说、有小说、无最近写作、搜索无结果等状态下的界面表现。
- 明确所有业务数字和项目内容只来自 SQLite 或真实应用状态。
- 保持首次使用路径直接：新建小说、导入小说。
- 保持实现依赖简单，不为主界面新增第三方 UI 依赖。

**Non-Goals:**

- 不实现新手向导、教程轮播或示例项目。
- 不设计复杂 AI 配置流程，只保留真实配置入口。
- 不引入远程同步、账号体系、云端模板市场。
- 不改变视频、富文本、托盘等非主界面能力。

## Decisions

1. **主界面使用同一布局骨架覆盖空状态和有数据状态。**
   - 选择：无小说时保留顶部、欢迎区、主卡片、统计卡和项目区，只替换内容为空状态。
   - 原因：创建第一本小说前后界面稳定，不像跳到另一个产品。
   - 替代方案：单独做一个 onboarding 页面。拒绝，因为用户明确暂不需要向导。

2. **业务内容只读真实数据，缺失时显示空状态。**
   - 选择：小说数量、总字数、今日目标、最近写作、项目列表全部来自 SQLite 或真实设置。
   - 原因：用户明确不要假配置和硬编码业务数据。
   - 替代方案：使用 demo 数据或默认项目。拒绝，因为会污染真实工作台判断。

3. **无小说时主卡片变成首次行动入口。**
   - 选择：主卡片显示“还没有小说项目”，提供“新建小说”和“导入小说”。
   - 原因：保留视觉重心，同时直接给出可执行动作。
   - 替代方案：隐藏主卡片只显示空列表。拒绝，因为首屏会显得塌陷。

4. **项目搜索只在存在真实小说时显示。**
   - 选择：小说列表非空时显示搜索、导入、新建；小说列表为空时不显示搜索框。
   - 原因：搜索空集合没有价值，也会让首次状态显得像坏掉。
   - 替代方案：永远显示搜索框。拒绝，增加无效控件。

5. **视觉实现走 Flutter 原生组件和本地资源。**
   - 选择：使用 Flutter 布局、颜色 token、卡片、图标按钮和可选本地图片/渐变封面。
   - 原因：主界面不需要 Canvas、WebGL 或新增 UI 包。
   - 替代方案：引入复杂动效或第三方 dashboard 组件。拒绝，收益低。

## Data Model Assumptions

后续实现可以从现有或待建 SQLite 表得到以下真实字段；如果字段暂不存在，任务应先补最小查询模型，而不是在 UI 写假数据。

- `novels`: 小说项目列表、标题、摘要、状态、封面路径或封面为空。
- `chapters` 或正文统计来源：每本小说和全局总字数。
- `writing_goal`: 今日目标字数；未设置时为空。
- `recent_writing`: 最近打开或最近编辑的小说/章节。

## Dashboard States

```text
novelCount == 0
  -> 首次空状态
  -> 主卡片：还没有小说项目
  -> 统计：项目 0、总字数 0、目标按真实设置显示
  -> 项目区：空状态行动

novelCount > 0 && recentWriting == null
  -> 有项目但无最近写作
  -> 主卡片：选择一本小说继续
  -> 项目区：显示真实项目列表和搜索

novelCount > 0 && recentWriting != null
  -> 常规工作台
  -> 主卡片：继续最近写作
  -> 项目区：显示真实项目列表和搜索

searchQuery != "" && searchResultCount == 0
  -> 搜索空状态
  -> 保留搜索框，显示“没有匹配的小说项目”
```

## Layout Notes

- 桌面宽屏：内容居中，最大宽度约 1200px；主区域左侧约 60%，右侧约 40%。
- 窄宽度：主卡片、统计卡、项目区纵向排列。
- 色彩：浅背景、白色卡片、浅灰边框、深色主按钮、红色品牌点缀。
- 卡片：圆角约 10-12px，轻边框，少阴影。
- 文案：短句，不解释功能机制，不放教程式段落。

## Risks / Trade-offs

- [Risk] 当前数据表可能还不存在或字段不完整 → 后续实现先补最小本地查询接口，再接 UI。
- [Risk] 无封面时卡片视觉过空 → 使用稳定的本地默认视觉样式，但不能伪造业务封面或小说内容。
- [Risk] 统计查询可能阻塞首屏 → 统计聚合应通过异步加载，加载中显示 skeleton 或空值占位。
- [Risk] “AI 配置”入口没有真实配置页 → 只保留入口到真实页面；页面不存在时任务应创建最小真实设置页或暂不显示入口。
