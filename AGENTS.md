# AGENTS.md

## 项目技术栈

- Flutter + Dart 的 Windows 桌面应用。
- 本地持久化使用 SQLite。
- 桌面集成包含本地窗口和托盘插件。
- 视频能力使用 media_kit，并需要 mpv 能力。
- 富文本需要支持 Markdown 和数学公式渲染。

## 前端 UI 规则

- 设计或修改前端界面前，先读取 `docs/ui-design-dna.md`。
- 项目主界面是视觉基准；新界面必须继承主界面的白底、细边框、克制红色、轻阴影、工具化密度，不要另起一套弹窗或表单风格。
- 所有 UI 布局改动交付前必须截图验证；检查桌面默认窗口尺寸和相关弹窗/下拉/滚动状态，确认无文本溢出、控件重叠、内容遮挡、错误裁切。
- 禁止交付存在重叠事故的 UI。发现重叠、遮挡或溢出时，必须先修正并重新截图验证。
- 用户验收通过的设计要素，后续必须沉淀回 `docs/ui-design-dna.md`：自行判断它属于通用规则、组件规则、功能区规则或禁用规则，并标明适用范围。
- 沉淀 UI 规则时只记录可复用、可执行的设计约束；不要把一次性实现细节、临时妥协或未验收尝试写进纲领。

## 运行态同步

- 修改 Flutter/Dart 代码后，必须确保用户实际通过 `一键启动.cmd` 打开的 Windows 桌面应用也更新到最新代码；不要只验证 web preview 或测试环境。
- 如果应用由 `flutter run` 启动且会话可用，优先执行热重载；涉及入口、状态初始化、路由、资源或热重载不生效时执行热重启。
- 如果用户通过 `一键启动.cmd` 或已构建 exe 验收，改动后必须重新构建 Windows Debug exe（例如 `flutter build windows --debug --no-pub`），并重启 `build/windows/x64/runner/Debug/ai_novel_factory.exe` 后再交付。
- 交付前说明已同步到哪种运行态：热重载、热重启或重建并重启 exe。

## 真实性约束

- 禁止生成假配置、假密钥、假路径、假接口地址、假数据文件或只为通过检查存在的占位产物。
- 禁止把环境相关值、业务参数、文件路径、端口、模型名、密钥名等写成无来源硬编码。
- 缺少真实值时，使用明确的配置入口、启动参数或文档化 TODO；会阻塞运行的必须直接说明，不要伪造默认值。
